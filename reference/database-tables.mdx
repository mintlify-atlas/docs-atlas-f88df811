## `equipos`

Inventory of all physical equipment pieces. Each piece belongs to a room and has a brand.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| `id` | `int(11)` | No | AUTO_INCREMENT | Primary key |
| `tipo` | `varchar(100)` | No | вЂ” | Equipment type (e.g. "Laptop", "Proyector") |
| `idmarca` | `int(11)` | No | вЂ” | Foreign key в†’ `marcas.id` |
| `codigo` | `varchar(100)` | No | вЂ” | Asset/inventory code |
| `idsala` | `int(11)` | No | вЂ” | Foreign key в†’ `salas.id` |
| `fechaingreso` | `date` | No | вЂ” | Date the equipment was registered |
| `estado` | `int(11)` | No | `1` | Operational status (see below) |

**Primary key:** `id`  
**Foreign keys:** `idmarca` в†’ `marcas.id`, `idsala` в†’ `salas.id`

### `estado` values

`estado` is an integer used as a boolean flag:

| Value | Meaning |
|-------|---------|
| `1` | Operational (Funcionamiento) |
| `0` | Under maintenance (Mantenimiento) |

<Note>
  When a maintenance record is created for a piece of equipment, `estado` should be set to `0`. When maintenance is closed, it should be restored to `1`.
</Note>

## `mantenimientos`

Maintenance records linking equipment to the technician who performed the work. Each record represents a single maintenance event.

| Column | Type | Nullable | Default | Description |
|--------|------|----------|---------|-------------|
| `id` | `int(11)` | No | AUTO_INCREMENT | Primary key |
| `idequipo` | `int(11)` | No | вЂ” | Foreign key в†’ `equipos.id` |
| `tipomantenimiento` | `int(11)` | No | вЂ” | Maintenance type (see below) |
| `problema` | `varchar(100)` | No | вЂ” | Brief description of the problem |
| `fechainicio` | `date` | No | вЂ” | Date maintenance started |
| `idmonitor` | `int(11)` | No | вЂ” | Foreign key в†’ `monitores.id` |
| `fechafin` | `date` | **Yes** | `NULL` | Date maintenance was completed |
| `descripcion` | `text` | **Yes** | `NULL` | Detailed description of work performed |

**Primary key:** `id`  
**Foreign keys:** `idequipo` в†’ `equipos.id`, `idmonitor` в†’ `monitores.id`

### `tipomantenimiento` values

`tipomantenimiento` is an integer enum:

| Value | Meaning |
|-------|---------|
| `1` | Preventivo вЂ” scheduled/preventive maintenance |
| `2` | Correctivo вЂ” corrective maintenance in response to a failure |

### Open vs. closed maintenance

| Field | Open record | Closed record |
|-------|-------------|---------------|
| `fechafin` | `NULL` | Date of completion |
| `descripcion` | `NULL` | Work description |

<Warning>
  A `NULL` value in `fechafin` means the maintenance event is still **in progress**. Do not interpret it as an error. The `descripcion` field is only populated when the record is closed.
</Warning>

## Prerequisites

Before you begin, make sure you have the following installed:

- **PHP 8.2 or later** with the `mysqli` extension enabled
- **MySQL 5.7+ or MariaDB 10.x**
- **Apache or Nginx** web server (or PHP's built-in server for local testing)
- A MySQL client such as phpMyAdmin or the `mysql` CLI

## Setup

<Steps>
  <Step title="Download the project">
    Clone or download the project files to your web server's document root.

    ```bash
    git clone https://github.com/LeynerDurk/Proyecto-pagina.git
    cd Proyecto-pagina
    ```

    If you received a ZIP archive, extract it to your web root (e.g., `/var/www/html/mantenimientos` or `C:\xampp\htdocs\mantenimientos`).
  </Step>

  <Step title="Create the database">
    Log in to MySQL and create a new database named `mantenimientos`.

    ```sql
    CREATE DATABASE mantenimientos CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    ```
  </Step>

  <Step title="Import the SQL schema">
    Import the bundled SQL dump to create all tables and sample data.

    ```bash
    mysql -u root -p mantenimientos < "mantenimientos (1).sql"
    ```

    <Note>
      The SQL file includes the full schema and any seed data. Run this command from the project root directory.
    </Note>
  </Step>

  <Step title="Configure the database connection">
    Open `config/database.php` and update the credentials to match your environment.

    ```php
    <?php
    $host     = 'localhost';
    $user     = 'root';       // your MySQL username
    $password = '';           // your MySQL password
    $database = 'mantenimientos';

    $dbConnection = mysqli_connect($host, $user, $password, $database);

    if (!$dbConnection) {
        echo 'Error en la conexión';
    }
    ```

    <Warning>
      Do not use the `root` user with no password in production. Create a dedicated MySQL user with access limited to the `mantenimientos` database.
    </Warning>
  </Step>

  <Step title="Start the application">
    Point your web server at the project directory, or use PHP's built-in server for quick local testing.

    ```bash
    php -S localhost:8000
    ```

    Open your browser and navigate to `http://localhost:8000`. You should see the brand management page.
  </Step>
</Steps>

## First actions

Follow these steps to set up the system with your first equipment record.

<Steps>
  <Step title="Create a brand">
    On the home page (`index.php`), enter a brand name such as `Dell` and click **Guardar**. Brands are required before you can register equipment.
  </Step>

  <Step title="Create a location (sede)">
    Navigate to **Sedes** in the top menu. Enter a location name such as `Bicentenario` and save it.
  </Step>

  <Step title="Create a room (sala)">
    Navigate to **Salas**. Enter a room name and select the sede you just created from the dropdown, then save.
  </Step>

  <Step title="Add a technician (monitor)">
    Navigate to **Monitores**. Enter the technician's full name and save. You will need at least one technician before logging maintenance jobs.
  </Step>

  <Step title="Register equipment">
    Navigate to **Equipos**. Fill in the equipment type (e.g., `Laptop`), select the brand and room, set the entry date, and save. The equipment is now registered and ready for maintenance tracking.
  </Step>

  <Step title="Log a maintenance job">
    Navigate to **Mantenimientos**. Fill in the form at the top of the page: select the equipment, choose the maintenance type (Preventivo or Correctivo), describe the problem, set the start date, and assign a technician, then click **Guardar**.
  </Step>
</Steps>

<Tip>
  To view the full maintenance history for any piece of equipment, go to **Equipos** and click the **Ver** button on the equipment's row.
</Tip>
